### DAgger v0

현재 DAgger v0의 의미:

- rollout execute policy는 learner다.
- expert는 같은 state에서 action proposal을 생성한다.
- replay에는 executed action, learner action, expert action을 저장한다.
- learner update는 `expert_actions`를 target으로 하는 BCFlow BC update다.
- intervention gate는 아직 꺼져 있다.

현재 config 핵심:

```text
actors.learner.kind = bc_flow
actors.learner.update.target_action_key = expert_actions
actors.expert.kind = rlpd
intervention.enabled = false
intervention.expert_query = always
intervention.gate.kind = always_off
training.action_mode = first_action
```

### Expert-Q gap intervention gate

2026-05-21에 `expert_q_gap` gate를 추가했다. 이 gate는 expert가 action-value critic을 가지고 있을 때
다음 score로 intervention trigger를 건다.

```text
q_gap = Q_expert(s, a_expert) - Q_expert(s, a_learner)
signal = q_gap > threshold
```

동작:

- signal이 켜졌을 때만 `intervention_prob` 확률로 expert intervention을 시작한다.
- `p_off`는 두지 않는다.
- intervention이 시작되면 `intervention_horizon` env step 동안 expert가 연속 제어한다.
- ensemble Q는 `q_agg=min|mean|max`로 scalar화한다.

중요한 구현 원칙:

- `expert_q_gap`은 RLPD 전용이 아니다.
- gate는 expert 내부의 `critic`, `q`, `qf` module 이름을 직접 탐색하지 않는다.
- gate는 `evaluate_q(obs, action, q_agg=...)` 또는 `q_values(obs, action, q_agg=...)`만 호출한다.
- multi-Q head shape과 `min|mean|max` aggregation은 expert agent/adapter 책임이다.
- 현재 RLPD/SAC agent에는 `evaluate_q_heads()`와 `evaluate_q(q_agg=...)`를 추가했다.
- SAC, TD3-BC 등도 같은 API를 맞추면 같은 gate를 쓸 수 있다.
- PPO처럼 V-only critic만 있는 expert는 이 gate를 바로 쓸 수 없다. action-value head 또는 adapter가 필요하다.
- `AgentPolicyView`는 stochastic agent의 `sample_actions_with_log_prob()`뿐 아니라 deterministic agent의 `sample_actions()`도 지원한다. 이 경우 log-prob은 NaN으로 둔다.

### Replay / buffer

현재 buffer 방향:

- `online`: learner rollout에서 얻은 모든 transition.
- `demo`: success-only demo 또는 좋은 online episode를 넣을 후보.
- `intervention`: expert intervention이 포함된 suffix/episode를 넣을 후보.

현재 구현/결정:

- online buffer에는 success 여부와 관계 없이 transition을 넣는다.
- demo insert mode는 `none`, `append`, `replace_worst` 계열로 확장 가능하게 둔다.
- n-step boundary는 기본적으로 qc_base처럼 boundary crossing sample을 drop하는 방향이다.
- timeout bootstrap/truncate mode는 value/RL에서 중요하므로 후속 작업으로 남겨둔다.
- image replay는 env/replay 인프라만 있고, policy/network image input은 아직 미정이다.

- `il/buffers/replay_buffer.py`
