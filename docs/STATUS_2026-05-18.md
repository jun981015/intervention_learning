## Intervention Pipeline

한 step에서 learner와 expert action을 모두 뽑고, gate가 실제 실행 action을 고른다.
replay에는 세 action을 모두 분리해서 저장한다.

```python
learner_output = learner.sample_action(obs, rng=learner_rng)
expert_output = expert.sample_action(obs, rng=expert_rng)
decision = gate.decide(step, obs, learner_output, expert_output, gate_rng)
action = expert_output.action if decision.use_expert else learner_output.action
```

buffer는 역할별로 나눈다.

- `online_buffer`: 모든 online transition
- `demo_buffer`: intervention 없이 성공한 trajectory 또는 offline demo
- `intervention_buffer`: first intervention 이후 suffix

failed expert intervention suffix는 `include_failed_interventions` flag로 포함 여부를 제어한다.
자세한 step logic과 routing은 [PIPELINE.md](PIPELINE.md)를 본다.

## DAgger Baseline

DAgger는 gate를 쓰지 않는다. learner action으로 env를 밟고, expert action은 같은 state의 BC label로만
저장한다. 별도 DAgger buffer 없이 `online_buffer`를 쓰면 된다. 따라서 `actions`는 learner 실행
action이고, `expert_actions`는 relabel target이다.
`DAggerConfig.store_expert_action=True`가 기본값이며, false로 두면 expert action 대신 NaN placeholder를
저장한다.
자세한 내용은 [DAGGER_BASELINE.md](DAGGER_BASELINE.md)를 본다.

## Replay와 Update 결정

- n-step backup API는 `ReplayBuffer.sample_sequence(batch_size, n, gamma)` 형태로 유지한다.
- qc_base 방식에 맞춰 n-step horizon 안에서 episode boundary를 넘는 sequence는 샘플에서 제외한다.
- 나중에 boundary에서 잘라 n-step 길이를 줄이고 bootstrap하는 variant를 비교할 수 있게 문서에 기록했다.
- `frame_stack`은 기본값 1이다.
- UTD는 config로 받게 했다.
- TD target에서는 target critic 앞 2개를 쓰고, 기본 aggregation은 `min`이다.
- actor update에서는 actor loss에 사용 가능한 critic 전체 평균을 쓴다.
- grad clip 기본값은 `None`이다.

세부 수식과 schema는 [REPLAY_AND_UPDATES.md](REPLAY_AND_UPDATES.md)를 본다.

구현된 generic wrapper:

- `il.policies.RLPDPolicy.from_checkpoint(...)`
- `il.policies.BCFlowPolicy.from_checkpoint(...)`

둘 다 `PolicyOutput(action, log_prob, info)`를 반환하므로 `choose_action()`에 같은 방식으로 넣을 수 있다.

## 오늘 마무리 기록

오늘 마지막 결정은 DAgger baseline을 별도 buffer 없이 `online_buffer`만 쓰는 방식으로 고정한 것이다.
rollout은 learner action으로만 진행하고, config가 허용할 때 expert action을 같은 transition의
`expert_actions`에 저장한다.

현재 DAgger 기본값:

```text
DAggerConfig.store_expert_action=True
actions = learner_action
learner_actions = learner_action
expert_actions = expert_action
controller_ids = LEARNER
interventions = 0
BC target_action_key = "expert_actions"
```

`store_expert_action=False`는 나중에 update-time relabeling ablation을 위한 옵션이다. 이 경우 rollout
중 expert를 호출하지 않고 `expert_actions`에 NaN placeholder를 저장한다.

마지막으로 확인한 검증:

```bash
conda run -n il python -m compileall -q il scripts
CUDA_VISIBLE_DEVICES=0 XLA_PYTHON_CLIENT_PREALLOCATE=false conda run -n il python scripts/smoke_test.py
```
