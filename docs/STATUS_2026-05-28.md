## Gate contract 정리

- `ControllerGate`는 `@runtime_checkable` Protocol이다.
- `TrainContext.gate`와 `build_gate()`는 `ControllerGate | None` 타입을 쓴다.
- build된 gate는 Protocol runtime check로 검증한다.
- `RandomGate`도 no-op `reset_episode()`를 구현해서 episode boundary에서 모든 gate를 같은 방식으로 reset할 수 있다.

ABC 부모 클래스로 강제하지 않고 Protocol을 유지했다. 이 레포의 현재 스타일은 structural typing에 가깝고,
새 gate는 `decide(...)`와 `reset_episode()` 모양만 맞으면 gate로 쓸 수 있다.

## Runtime config 반영

다음 public config가 runtime에 반영된다.

- `training.initial_collect.unit`: `steps` 또는 `episodes`만 허용한다.
- `training.update_interval`: 지정한 env step 간격에서만 update한다.
- `training.updates_per_step`: 한 env step에서 update spec 전체를 여러 번 실행하고 metric은 평균으로 기록한다.
- `checkpointing.save_final`: final checkpoint 저장 여부를 제어한다.
- `evaluation.render_video`, `evaluation.video_episodes`, `evaluation.video_frame_skip`: eval video 저장을 제어한다.

`keep_last`는 사용자가 필요 없다고 판단해서 구현하지 않았다. `storage.store_*`는 아직 canonical replay schema를
끄고 켜는 runtime switch가 아니라 선언적 설정에 가깝다.

## Initial collect 동작

초기 수집 중에는 env step과 replay insert는 계속 수행하지만 gradient update는 돌리지 않는다.

- `steps`: `step >= count`가 되면 update 가능
- `episodes`: `episode_count >= count`가 되면 update 가능

사용자가 `BufferTooSmall` custom exception을 원하지 않아 제거했다. replay가 아직 sequence length보다 작을 때는
기존처럼 sampler의 `"smaller than sequence_length"` `ValueError` 메시지를 보고 train loop가 update를 skip한다.

## Evaluation / video

평가 코드는 두 층으로 나뉜다.

- `evaluate_policy(policy, env, ...)`: policy와 env만 있으면 평가 가능한 generic 함수
- `evaluate_context_policy(context, step=...)`: train loop에서 쓰는 context adapter

Video 저장은 `video_dir`와 `video_episodes > 0`이 있을 때만 동작한다. Video 저장을 요청했는데 env에
`render()`가 없거나 `render()`가 `None`을 반환하면 `ValueError`를 낸다. Metric episode와 video episode는
분리되어 있고, video episode는 return/length/success metric에 섞이지 않는다.

## Observation mode

`RobomimicLowdimWrapper`에 state-only dict mode가 추가됐다.

```yaml
env:
  observation_mode: state
```

이 모드는 `{"state": low_dim_state}`를 반환한다. 기존 `lowdim`은 flat array를 유지한다. Image와
`pixels_state` env/replay plumbing은 있지만, 현재 trainable policy들은 아직 low-dim network만 지원한다.

## Residual RL 상태

현재 residual 계열은 두 가지다.

- `residual_rlpd`: SAC/RLPD-style stochastic residual actor (`TanhNormal`)
- `residual_td3`: deterministic TD3-style residual actor. tanh actor head, target actor, delayed actor update,
  target policy smoothing, critic-only warmup을 지원하고 SAC entropy/alpha path는 없다.

`rlpd`, `residual_rlpd`, `residual_td3` critic loss는 valid sample 수 기준으로 normalization하도록 정리했다.
valid fraction이 낮아질 때 loss scale이 조용히 줄어드는 문제를 줄이기 위한 변경이다.

## Residual composition + intervention gate

Residual action composition과 intervention gate를 함께 쓸 수 있게 연결했다.

```yaml
training:
  action_composition: residual
intervention:
  enabled: true
```

이 조합에서는 내부적으로 `rollout.execute = "gate"`가 되지만, gate가 보는 learner proposal은 raw residual이 아니라
실제 실행 가능한 full action이다.

```text
learner.action = clip(base_action + residual_scale * raw_residual)
```

Gate가 expert를 선택해도 replay에는 residual metadata를 저장한다.

```text
base_actions      = current obs의 base policy action
next_base_actions = next obs의 base policy action
residual_actions  = executed_action - base_actions
```

따라서 expert-selected intervention step에서 `residual_actions`는 frozen base policy 위에서 expert action을
재현하기 위한 residual correction target으로 해석된다.

Build/eval도 이 조합에 맞췄다.

- `il.train.build_context()`는 `rollout.execute`가 `gate`여도 `action_composition: residual`이면 `actors.base`를 build한다.
- context eval은 gate가 아니라 learner의 base+residual policy를 평가한다.
