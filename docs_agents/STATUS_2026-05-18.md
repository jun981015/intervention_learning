## Intervention Pipeline

Every environment step samples both learner and expert actions. The gate decides which action is executed.
Replay stores learner proposal, expert proposal, executed action, and gate metadata separately.

```python
learner_output = learner.sample_action(obs, rng=learner_rng)
expert_output = expert.sample_action(obs, rng=expert_rng)
decision = gate.decide(step, obs, learner_output, expert_output, gate_rng)
action = expert_output.action if decision.use_expert else learner_output.action
```

Role-specific buffers:

- `online_buffer`: all online transitions.
- `demo_buffer`: autonomous successful trajectories or offline demos.
- `intervention_buffer`: suffix after the first intervention.

Failed expert intervention suffixes are controlled by the `include_failed_interventions` flag.
See `docs_agents/PIPELINE.md`.

## DAgger Baseline

DAgger does not use gating. The learner action advances the environment, and the expert action is
stored only as the BC relabel target for the same state. No separate DAgger buffer is needed; use
`online_buffer`. Therefore `actions` is the executed learner action and `expert_actions` is the
supervised target. `DAggerConfig.store_expert_action=True` is the default; false stores a NaN
placeholder instead. See `docs_agents/DAGGER_BASELINE.md`.

## Replay and Update Decisions

- Keep the n-step API as `ReplayBuffer.sample_sequence(batch_size, n, gamma)`.
- Match the QC reference behavior: sequences crossing an episode boundary inside the n-step horizon are excluded.
- A future variant can compare boundary truncation with shorter n-step bootstrap.
- `frame_stack` defaults to 1.
- UTD is configurable.
- TD target uses the first two target critics. Default target Q aggregation is `min`.
- Actor updates average over all available critics.
- Default grad clipping is `None`.

See `docs_agents/REPLAY_AND_UPDATES.md`.

## End-of-day Notes

The final DAgger decision today is to avoid a separate DAgger buffer. Use only `online_buffer`.
Rollouts execute learner actions. If configured, expert actions are stored in the same transition as
`expert_actions`.

Current DAgger default:

```text
DAggerConfig.store_expert_action=True
actions = learner_action
learner_actions = learner_action
expert_actions = expert_action
controller_ids = LEARNER
interventions = 0
BC target_action_key = "expert_actions"
```

`store_expert_action=False` is reserved for a future update-time relabeling ablation. In that mode,
rollout does not call the expert and stores a NaN placeholder in `expert_actions`.
