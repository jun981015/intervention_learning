# Status 2026-05-21

This is the current implementation snapshot for `intervention_learning`.
The human-facing Korean version is `docs/STATUS_2026-05-21.md`.

## Project Direction

`qc` and `qc_base` are references only. The actual project is `intervention_learning`.
The immediate goal is a minimal Robomimic Square pipeline that loads a learner policy and an expert policy, rolls out the learner, stores expert relabels, and updates the learner with DAgger/BC-style training.

### DAgger v0

Current DAgger v0 behavior:

- Learner executes rollouts.
- Expert is queried on the same states and its action is stored.
- Replay stores executed action, learner action, and expert action.
- Learner update is BCFlow BC using `expert_actions` as the target.
- Intervention gate is currently disabled.

Important config values:

```text
actors.learner.kind = bc_flow
actors.learner.update.target_action_key = expert_actions
actors.expert.kind = rlpd
intervention.enabled = false
intervention.expert_query = always
intervention.gate.kind = always_off
training.action_mode = first_action
```

### Expert-Q gap intervention gate

Added on 2026-05-21. This gate starts expert intervention when the expert's
action-value critic prefers the expert proposal over the learner proposal:

```text
q_gap = Q_expert(s, a_expert) - Q_expert(s, a_learner)
signal = q_gap > threshold
```

Behavior:

- If the signal is true, start intervention with probability `intervention_prob`.
- There is no `p_off` in this design.
- Once started, expert control continues for `intervention_horizon` env steps.
- Q ensembles are reduced with `q_agg=min|mean|max`.

Implementation constraints:

- Do not hardcode this gate to `kind == "rlpd"`.
- Do not inspect expert internals such as `critic`, `q`, or `qf` module names.
- The gate only calls `evaluate_q(obs, action, q_agg=...)` or `q_values(obs, action, q_agg=...)`.
- Multi-Q head shapes and `min|mean|max` aggregation are expert agent/adapter responsibilities.
- Current RLPD/SAC agent exposes `evaluate_q_heads()` and `evaluate_q(q_agg=...)`.
- SAC/TD3-BC-style experts can use the same gate by matching this API.
- PPO value-only experts cannot use this gate without an action-value adapter/head.
- `AgentPolicyView` supports both stochastic `sample_actions_with_log_prob()` and deterministic `sample_actions()`.

Files:

- `il/gating/expert_q_gap.py`
- `il/algo/rl/rlpd.py`
- `il/policies/agent_view.py`
- `config/expert_q_gap_square.yaml`

Validated with compile checks, a generic fake-Q smoke test, config load smoke, and `git diff --check`. Still needs real Robomimic Square rollout smoke with a restored pretrained expert.

Rules:

- For the RLPD expert, keep `network: {}` in YAML so the artifact config is used.

### Replay / buffers

Buffers:

- `online`: all learner rollout transitions.
- `demo`: success-only demos or good online episodes.
- `intervention`: expert intervention suffixes/episodes.

Current decisions:

- Online buffer receives transitions regardless of success.
- Demo insert mode can be `none`, `append`, or replacement-style later.
- N-step boundary behavior currently follows qc_base-style drop.
- Timeout bootstrap / truncate mode is a later value/RL TODO.
- Image replay infrastructure exists, but image policy training is not implemented yet.

### Logger

Current logger behavior:

- Records metrics every env step.
- Flushes one averaged row every `log_interval`.
- Losses and gradients are interval averages.
- Replay size, episode count, recent env metrics, and routing metrics keep the last value.
- Adds `train/log_interval_records` to each flushed row.
- Eval metrics use immediate logging and do not mix with the train accumulator.
