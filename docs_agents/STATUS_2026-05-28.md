## Git And Artifact Policy

- `figures/` is now ignored in `.gitignore`; it is intended for generated plots.
- Checkpoints, replay buffers, videos, logs, W&B files, and experiment outputs
  remain out of git.
- The latest pushed commit on `main` is `71294fe`.

## Gate Contract Cleanup

- `ControllerGate` is now `@runtime_checkable`.
- `TrainContext.gate` and `build_gate()` are typed as `ControllerGate | None`.
- Built gates are validated with the Protocol at construction time.
- `RandomGate` implements no-op `reset_episode()`, so the train loop can reset
  any gate uniformly at episode boundaries.

The project still uses a Protocol rather than an abstract base class. That is
intentional: gates are accepted structurally when they expose the required
`decide(...)` and `reset_episode()` methods.

## Runtime Config Wiring

The public config now reaches runtime for these fields:

- `training.initial_collect.unit`: must be `steps` or `episodes`; other values
  raise `ValueError`.
- `training.update_interval`: train updates only on matching env steps.
- `training.updates_per_step`: runs repeated configured update specs and logs
  averaged update metrics.
- `checkpointing.save_final`: controls final checkpoint saving.
- `evaluation.render_video`, `evaluation.video_episodes`, and
  `evaluation.video_frame_skip`: control optional video rollout.

`keep_last` was intentionally not wired because the user decided it is not
needed. `storage.store_*` remains mostly declarative and is not a runtime switch
for the canonical replay schema.

## Initial Collection Behavior

Initial collection is controlled by `train.initial_collect_unit` and
`train.initial_collect_count`. While initial collection is active, environment
steps still populate replay but gradient updates do not run.

Supported units:

- `steps`: updates can start when `step >= count`.
- `episodes`: updates can start when `episode_count >= count`.

The custom `BufferTooSmall` exception was not added. Underfilled replay still
uses the existing train-loop skip path for the replay sampler's
`"smaller than sequence_length"` `ValueError` text.

## Evaluation And Video

Evaluation was split into two layers:

- `evaluate_policy(policy, env, ...)`: generic evaluation that only needs a
  policy and an env. It can optionally save videos.
- `evaluate_context_policy(context, step=...)`: adapter used by the train loop.

Video behavior:

- Video is saved only when `video_dir` and `video_episodes > 0` are provided.
- Video rollout calls `env.render()` and raises `ValueError` if the env does not
  provide render frames.
- Metric episodes and video episodes are separate; video episodes are not mixed
  into return/length/success metrics.
- The env builder enables `render_offscreen=True` for the eval env only when
  video rendering is requested.

## Observation Modes

`RobomimicLowdimWrapper` now supports a state-only dict mode:

```yaml
env:
  observation_mode: state
```

This returns observations as `{"state": low_dim_state}`. The existing `lowdim`
mode still returns the legacy flat array. Image and `pixels_state` plumbing
remain available in env/replay, but current trainable policies are still
low-dimensional and do not have image encoders.

## Residual RL State

Implemented residual agents:

- `residual_rlpd`: SAC/RLPD-style stochastic residual actor using `TanhNormal`.
- `residual_td3`: deterministic TD3-style residual actor with tanh output,
  target actor, delayed actor updates, target policy smoothing, critic-only
  warmup, and no SAC entropy/alpha path.

Critic loss normalization in `rlpd`, `residual_rlpd`, and `residual_td3` now
normalizes by valid samples instead of silently scaling down with invalid n-step
windows.

## Residual Composition With Intervention Gates

Residual action composition and intervention gates can now be used together.
When config has:

```yaml
training:
  action_composition: residual
intervention:
  enabled: true
```

`new_schema_to_legacy_recipe()` sets `rollout.execute = "gate"`, but rollout
still builds the learner proposal as a full executable action:

```text
learner.action = clip(base_action + residual_scale * raw_residual)
```

The gate compares that full learner action against the expert action. If the
gate chooses the expert, the env executes the expert action, but replay still
stores residual metadata:

```text
base_actions      = base policy action at current obs
next_base_actions = base policy action at next obs
residual_actions  = executed_action - base_actions
```

For expert-selected intervention steps, `residual_actions` therefore represents
the residual correction that would reproduce the executed expert action from the
frozen base policy. This is the right target shape for residual BC/RL updates.

Build and eval were also updated for this case:

- `il.train.build_context()` builds `actors.base` whenever residual composition
  is enabled, even if `rollout.execute` is `gate`.
- Context eval evaluates the learner's base+residual policy, not the gate.
